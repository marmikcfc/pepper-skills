# Messaging Framework

Build the foundational messaging architecture — from category definition down to proof points and objection responses.

## When to Use
- "We need a messaging framework"
- "Help us clarify how to talk about our product"
- "Our messaging is inconsistent across channels"
- "Prepare messaging for [launch/rebrand]"

## Prerequisites
- `ANTHROPIC_API_KEY`
- ICP, product description, competitive context

## Workflow

**Step 1: Define the category**

- What category does your product belong to?
- Are you creating a new category or competing in an existing one?
- What's the category's job to be done?

**Step 2: Write the positioning statement**

> For [target customer] who [have this problem], [product] is a [category] that [key benefit]. Unlike [competitive alternative], we [key differentiator].

Write 3 variants. Test against: Is this true? Is this different? Does our best customer recognize themselves?

**Step 3: Define value pillars**

| Pillar | Claim | Proof point | Customer quote |
|--------|-------|-------------|----------------|
| | | | |

**Step 4: Build the message hierarchy**

| Level | Message | Used in |
|-------|---------|---------|
| Tagline | 5-7 word brand promise | Homepage hero, logo lockup |
| Elevator pitch | 1-2 sentences | Sales intro, bio |
| Value prop | 3-4 sentences | Homepage, pitch deck |
| Full narrative | 1-2 paragraphs | About page, media kit |

**Step 5: Tailor by buyer**

| Persona | Their concern | Message emphasis | Proof type |
|---------|--------------|------------------|-----------|

**Step 6: Build the objection map**

| Objection | Root concern | Response |
|-----------|-------------|----------|
| "We use [competitor]" | | |
| "Too expensive" | | |
| "No time to implement" | | |

**Step 7: Assemble the doc**

1. One-liner → 2. Positioning → 3. Value pillars → 4. Message by audience → 5. Channel variants → 6. Objections → 7. Words to use/avoid

## Output
Complete messaging framework with positioning statement, value pillars, hierarchy, audience variants, and objection map.
